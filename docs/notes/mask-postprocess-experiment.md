# Mask 后处理实验：失败诊断（2026-06-16）

分支 `feature/mask-postprocess` 给 `ClothingDetector` 加了 `clean_mask()`
基础后处理步骤，希望通过清理分割 mask 解决 V2/V3 实验中暴露的"mask 含阴影"问题。

## 流水线

```
_segment_edge() 输出 raw mask
  → 闭运算（5×5 椭圆核，填 Canny 抓出来的小洞）
  → connectedComponentsWithStats 取最大连通分量（去掉远处散点）
  → findContours+drawContours 填内部孔洞
  → 3×3 腐蚀 1 像素（去 Canny 边界 overshoot 撑出来的"光晕"外延）
```

## 验证

`tools/compare_mask_postprocess.py` 跑 3 张测试图 × 2 关键点方法（V1 geometric / V3 width）× 2 后处理开关 = 12 个组合。

### Mask 像素变化（衡量清理是否真的动了 mask）

| 图片 | raw px | clean px | diff | 占比 |
|---|---:|---:|---:|---:|
| 蓝旗袍 | 135781 | 133643 | 2138 | 1% |
| 浅粉 T 恤 | 590366 | 586081 | 4285 | **0.7%** |
| 白衬衫 | 371704 | 368338 | 3366 | 0% |

**所有 3 张图上，cleaned mask 与 raw mask 几乎一致**（差异在 1% 以内），且 y 范围没变（mask 仍然延伸到衣服底部的阴影区）。

## 结论

### 后处理对 V1 = 无影响（±2 像素）

V1 用固定比例带 [neck+12%, neck+22%] 找肩/腋，本来就不依赖 mask 形状。
cleaned mask 把 mask 边缘削了 1 像素，对落在 mask 内部的极值点没有影响。

### 后处理对 V3 = **毫无帮助**（仍错位 300+ 像素）

V3 找错位置的原因不是"mask 有小毛刺"或"远处散点"，而是 **mask 整体向下延伸到了衣服下方的阴影**，T 恤/白衬衫的阴影跟衣服是**同一连通分量**（1 个 CC），所以：
- **最大连通分量**：no-op，没东西可滤
- **填洞**：mask 内部本来就没洞
- **闭运算**：对连通的阴影没作用
- **边缘腐蚀 1px**：只削 1 像素的边界，对底部 200 像素的延伸完全无效

基础后处理只能处理"小毛刺"级污染，处理不了"主体延伸"级污染。

## V3 真正的解法（明确后续方向）

要让 V3 在 T 恤/白衬衫上工作，必须在 mask 阶段把"衣服本体"和"下方阴影"分开。两个可行方向：

1. **行向实心度（row solidity）剪枝**
   对每行 mask 算 longest_run / span，低于阈值（如 0.5）的行视为"碎片行"裁掉。
   阴影在 mask 底部通常分裂成多段（T 恤 y=900 处分 3 段），实心度会塌，
   借此把 mask 截短到衣服本体高度。

2. **亮度减阴影（HSV/V 通道阈值）**
   衣服本体亮度高、阴影亮度低。对每行算 HSV-V 中位数，
   找 V 突降的位置（如从 200 跌到 80）作为阴影起点，向下整段裁掉。

两条都比 clean_mask() 复杂，**单独开新分支做**。这次实验确认：
mask 后处理是**必要但不充分**的上游改进。

## 这次实验留下什么

- `ClothingDetector.clean_mask()` 静态方法保留，CLI `--mask-postprocess/--no-mask-postprocess` 开关保留（默认 True）。
- 对 V1 完全透明；对 V3 留作"基础卫生"——比没有强一点点，但解决不了根本问题。
- 关键诊断信息已经落盘到 `output/*/debug/clothing_1_mask_raw.png` 和 `clothing_1_mask.png`，可以直观对比 raw vs clean。
- 配套工具 `tools/compare_mask_postprocess.py` 留作以后复跑。
