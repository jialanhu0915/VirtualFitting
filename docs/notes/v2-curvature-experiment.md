# V2 曲率方法实验：失败诊断（2026-06-16）

分支 `feature/skeleton-symmetry-keypoints` 提交 `77b01bd` 实现了 V2 关键点
派生方法（`ClothingDetector(keypoint_method="skeleton")`），思路是用**轮廓
曲率 NMS + 对称轴 + 宽度谷值**三信号融合替代 V1 的"凹点 + 极值"几何
派生。三张测试图上的对比结果出乎意料——**V2 全部劣于 V1**。

## 三张图对比

| 图片 | V1 关键点 | V2 关键点 | 结论 |
|---|---|---|---|
| 蓝旗袍 | 8 点全对，腋下取到腰部收窄 | 肩/腋挤在肩部下方 28px 内 | V1 更准 |
| 浅粉 T 恤 | 8 点全对，肩在袖顶、腋在袖底内角 | 肩/腋挤在袖底 25px 内 | V1 完胜 |
| 白衬衫（V 形翻领） | 8 点全对 | 左右肩 y 差 77px、左右腋 y 差 296px | V1 完胜 |

可视化对照：`output/compare_{qipao,tshirt,whiteshirt}.jpg`。

## 失败根因

V2 假设"衣服轮廓的**结构转角**（肩峰、腋下 V、底摆角）= **高曲率点**"。
这个假设在两种衣服上不成立：

1. **平滑款**（旗袍、T 恤肩部、衬衫领圈）：肩部是 smooth curve，没有尖
   角，曲率小；T 恤的袖底外/内角才是真正的高曲率转角。
2. **结果**：V2 把"袖底外角"当肩、"袖底内角"当腋，本质上抓到的是
   袖子两个角，不是肩腋。

**结构关键点 ≠ 高曲率点**。肩/腋在物理上是"sleeve 与 body 的边界"，
轮廓上是宽度突变的位置，不一定伴随转角。

## 这次实验留下什么

- `ClothingDetector.keypoint_method` 参数保留，CLI `--method skeleton`
  仍然可用。
- 默认仍是 `geometric`（V1）。
- V2 调用的工具函数（`_find_symmetry_axis`、`_skeleton_features`、
  `_contour_curvature`、`_find_symmetric_pair`）保留，可能对将来其他
  场景有用（比如对称性校验、骨架分析）。
- 弃用曲率方法找肩腋的思路。**自适应的肩腋定位应该走宽度剖面
  （width(y) 的局部极值），不是曲率**——这是 V3 的方向。

## 后续方向

- **V3 宽度剖面法**：算每个 y 的 mask 宽度 → 平滑 → 用
  `scipy.signal.find_peaks` 找局部极大（肩/底摆）和极小（腋/腰）→
  按 y 区间分配。比 V1 固定比例带更稳，能适应长裙、宽 T、超短款等。
- **V1 + 小修补**：V1 已在 3 张图上稳定，V3 之前不必再改 V1。
