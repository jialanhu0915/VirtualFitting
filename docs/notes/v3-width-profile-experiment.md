# V3 宽度剖面法实验：失败诊断（2026-06-16）

分支 `feature/width-profile-keypoints` 实现了 V3 关键点派生方法
（`ClothingDetector(keypoint_method="width")`），思路是**用 mask 宽度剖面
的极值自适应定位肩/腋**，取代 V1 的固定比例带。

## 算法核心

```
widths[y] = mask 在 y 行的像素数
widths_s  = 高斯平滑 widths
shoulder_y = argmax(widths_s) in [y_min, y_min + 0.60*ch]
armpit_y   = argmin(widths_s) in [shoulder_y + 0.03*ch, y_min + 0.70*ch]
```

试过 `connectedComponents` 取最大分量去掉阴影，但 T 恤阴影跟衣服本体
是同一连通分量（只有 1 个连通域，bbox=1014×832 远超衣服），救不了。

## 三张图对比

| 图片 | V1 关键点 | V3 关键点 | 结论 |
|---|---|---|---|
| 蓝旗袍 | 8 点全对 | 8 点几乎全对（肩 122 / 腋 125，V1 是 125 / 398）| V3 在 qipao 上 shoulder 和 armpit 几乎重合（y 差 3px），但 qipao 无袖，"腋"语义本就是收窄的腰，V1 找到的 398 更合理 |
| 浅粉 T 恤 | 8 点全对（肩 420 / 腋 655）| 肩 736 / 腋 782（**mask 阴影扩展到 y=900，宽 900px 远超袖子**）| V3 严重错位 |
| 白衬衫 | 8 点全对（肩 250 / 腋 478）| 肩 548 / 腋 592（**mask 阴影同上**）| V3 严重错位 |

## 失败根因

V3 的"上半 argmax"假设**宽度曲线在肩部有局部极大值**。这个假设只在
**mask 干净**的衣服上成立。T 恤和白衬衫的 CLAHE+Canny 分割把**衣服下方的
阴影**也吃进 mask，连通成一个整体（只有 1 个连通分量）。于是：

- T 恤 mask 在 y=900 处分三段 (116,281)(313,895)(951,1103)，总宽 987px
  >> 真实衣身 582px
- 宽度曲线 y=420→900 一直增长，argmax 落在 y=736 而非 y=420

V1 反而能正确处理这种情况，**原因不是 V1 算法多聪明，而是它用固定比例
带 [neck+12%, neck+22%] 把搜索范围限制在 y=368~475，恰好绕开了阴影区**。
也就是说，V1 的"硬编码百分比"在这个具体数据集上是隐式的抗噪鲁棒性。

## 这次实验留下什么

- `ClothingDetector.keypoint_method` 参数保留，CLI `--method width` 仍可调用。
- `_extract_keypoints_width` 保留，含连通分量清洗 + 高斯平滑 + 调试可视化。
- 弃用"全图 argmax 当肩"策略。**V3 的方向（宽度剖面自适应）仍然是对的，
  但需要 (a) 更好的 mask 分割 或 (b) 限制搜索区间**。

## 后续方向

- **改进 mask 分割**（CLAHE 阈值化 + GrabCut 兜底 + 阴影 mask 减去）。
  这是上游问题，不在关键点模块内。
- **V3.1 = V3 + 限制区间**：上半 35% 内的 argmax（类似 V1 的窄带）。
  本质退化为 V1 的微调，意义不大。
- **V3.2 = 宽度导数法**：找 d(width)/dy 突变点（sleeve 边界），
  比 argmax 更鲁棒，需要更多测试。
- **保持 V1 默认**：三张图上 V1 都稳定，V3 失败时回退到 V1。
